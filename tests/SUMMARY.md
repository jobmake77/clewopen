## 任务概述
作为测试 Agent，负责为 OpenCLEW 项目创建完整的测试方案，验证文件下载、下载统计、评分统计等核心功能。

- 使用 pg 进行数据库验证
- 包含完整的测试流程:
  - 服务健康检查
  - 自动创建测试用户
  - 文件下载 API 测试
  - 下载统计验证
  - 评分统计验证

#### Bash 版本 (`run-tests.sh`)
- 使用 curl 进行 API 测试
- 使用 psql 进行数据库查询
- 彩色输出，易于阅读
- 与 Node.js 版本功能一致

## 发现的潜在问题

### 1. 下载 API 实现
**当前实现**: 返回 downloadUrl，由前端处理下载
**潜在问题**:
- 如果 package_url 是相对路径，可能无法访问
- 没有文件存在性验证

**建议**:
- 在下载 API 中验证文件是否存在
- 或者直接返回文件流而不是 URL

### 2. 评价触发器
**当前实现**: 使用数据库触发器更新评分
**潜在问题**:
- 只在 INSERT 和 UPDATE 时触发
- 如果评价被删除 (soft delete)，评分不会更新

**建议**:
- 添加 DELETE 触发器
- 或在触发器中检查 deleted_at 字段

### 3. 并发安全
**当前实现**: 使用触发器更新计数
**潜在问题**:
- 高并发下可能有性能问题

**建议**:
- 考虑使用 Redis 缓存计数
- 定期同步到数据库
