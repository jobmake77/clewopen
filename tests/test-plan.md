# OpenCLEW 测试计划

## 测试环境
- 后端: http://localhost:3001
- 前端: http://localhost:5173
- 数据库: PostgreSQL (Docker)

## 测试范围

### 1. 文件下载测试

#### 1.1 下载 API 测试
- **测试点**: POST /api/agents/:id/download
- **前置条件**: 用户已登录，Agent 存在
- **测试步骤**:
  1. 获取有效的 JWT token
  2. 调用下载 API
  3. 验证响应包含 downloadUrl
  4. 验证响应状态码为 200
- **预期结果**: 返回包含下载链接的 JSON 响应

#### 1.2 真实文件下载测试
- **测试点**: 验证返回的 downloadUrl 是否可访问
- **测试步骤**:
  1. 使用下载 API 获取 downloadUrl
  2. 使用 curl 访问该 URL
  3. 检查 Content-Type 响应头
  4. 检查 Content-Disposition 响应头
  5. 验证文件大小 > 0
- **预期结果**: 成功下载文件，响应头正确

#### 1.3 文件不存在测试
- **测试点**: 下载不存在的 Agent
- **测试步骤**:
  1. 使用无效的 Agent ID 调用下载 API
  2. 验证返回 404 错误
- **预期结果**: 返回 404 错误和错误消息

#### 1.4 未认证用户测试
- **测试点**: 未登录用户尝试下载
- **测试步骤**:
  1. 不携带 token 调用下载 API
  2. 验证返回 401 错误
- **预期结果**: 返回 401 未授权错误

### 2. 下载统计测试

#### 2.1 下载计数增加测试
- **测试点**: 验证 downloads_count 字段增加
- **测试步骤**:
  1. 查询 Agent 的初始 downloads_count
  2. 执行下载操作
  3. 再次查询 Agent 的 downloads_count
  4. 验证计数增加 1
- **预期结果**: downloads_count 正确增加

#### 2.2 下载记录创建测试
- **测试点**: 验证 downloads 表中创建记录
- **测试步骤**:
  1. 执行下载操作
  2. 查询 downloads 表
  3. 验证存在对应的下载记录
  4. 验证记录包含正确的 agent_id, user_id, version
- **预期结果**: 下载记录正确创建

#### 2.3 多次下载累加测试
- **测试点**: 验证多次下载计数累加
- **测试步骤**:
  1. 记录初始 downloads_count
  2. 执行 3 次下载操作
  3. 验证 downloads_count 增加 3
  4. 验证 downloads 表中有 3 条记录
- **预期结果**: 计数和记录数正确

#### 2.4 触发器测试
- **测试点**: 验证数据库触发器自动更新计数
- **测试步骤**:
  1. 直接向 downloads 表插入记录
  2. 查询 agents 表的 downloads_count
  3. 验证计数自动增加
- **预期结果**: 触发器正常工作

### 3. 评分统计测试

#### 3.1 评价提交测试
- **测试点**: POST /api/agents/:id/rate
- **测试步骤**:
  1. 提交评价 (rating: 5, comment: "测试评价")
  2. 验证返回成功响应
  3. 验证评价状态为 'pending'
- **预期结果**: 评价成功提交，等待审核

#### 3.2 评价记录验证测试
- **测试点**: 验证 reviews 表中的记录
- **测试步骤**:
  1. 提交评价
  2. 查询 reviews 表
  3. 验证记录存在且字段正确
  4. 验证 status = 'pending'
- **预期结果**: 评价记录正确创建

#### 3.3 评价批准后评分更新测试
- **测试点**: 验证批准评价后 Agent 评分更新
- **测试步骤**:
  1. 提交评价 (rating: 4)
  2. 手动更新评价状态为 'approved'
  3. 查询 Agent 的 rating_average
  4. 验证评分正确更新
- **预期结果**: rating_average 正确计算

#### 3.4 多个评价平均分计算测试
- **测试点**: 验证多个评价的平均分计算
- **测试步骤**:
  1. 提交 3 个评价 (rating: 5, 4, 3)
  2. 批准所有评价
  3. 验证 rating_average = 4.00
  4. 验证 reviews_count = 3
- **预期结果**: 平均分和计数正确

#### 3.5 重复评价防止测试
- **测试点**: 同一用户不能重复评价
- **测试步骤**:
  1. 提交第一次评价
  2. 尝试再次提交评价
  3. 验证返回 400 错误
- **预期结果**: 返回错误消息 "You have already reviewed this agent"

#### 3.6 评分范围验证测试
- **测试点**: 验证评分必须在 1-5 之间
- **测试步骤**:
  1. 尝试提交 rating: 0
  2. 尝试提交 rating: 6
  3. 验证都返回 400 错误
- **预期结果**: 返回错误消息 "Rating must be between 1 and 5"

#### 3.7 触发器测试
- **测试点**: 验证评价触发器自动更新评分
- **测试步骤**:
  1. 直接更新 reviews 表的 status 为 'approved'
  2. 查询 agents 表的 rating_average 和 reviews_count
  3. 验证自动更新
- **预期结果**: 触发器正常工作

### 4. 集成测试

#### 4.1 完整流程测试
- **测试步骤**:
  1. 用户注册/登录
  2. 浏览 Agent 列表
  3. 查看 Agent 详情
  4. 下载 Agent
  5. 提交评价
  6. 验证前端显示的数据
- **预期结果**: 整个流程顺畅，数据一致

#### 4.2 前端数据显示测试
- **测试点**: 验证前端显示的统计数据
- **测试步骤**:
  1. 在前端查看 Agent 卡片
  2. 验证显示的 downloads_count
  3. 验证显示的 rating_average
  4. 验证显示的 reviews_count
- **预期结果**: 前端数据与数据库一致

#### 4.3 并发下载测试
- **测试点**: 多用户同时下载
- **测试步骤**:
  1. 使用 3 个不同用户同时下载
  2. 验证 downloads_count 正确增加 3
  3. 验证没有数据竞争问题
- **预期结果**: 并发处理正确
